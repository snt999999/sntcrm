# SIGMA sender / имя отправителя

Менеджер SIGMA сообщил: на Tele2 подключено имя СОЛНЦАНЕТ, на остальных операторах SIGMA подменяет отправителя на общие имена AdvFactory, B-Media и т.д.

Поэтому в код добавлена безопасная логика:

1. Можно не передавать sender в SIGMA вообще — SIGMA сама подставит доступного отправителя по оператору.
2. Если указать SIGMA_SENDER=СОЛНЦАНЕТ или SOLNCANET, а SIGMA вернёт "Sender not found", система автоматически повторит отправку без sender.
3. SMS не будет падать окончательно только из-за неподходящего имени отправителя.

Рекомендуемые переменные Cloudflare:

SIGMA_API_TOKEN=токен_sigma
SIGMA_SENDER_MODE=auto

Переменную SIGMA_SENDER лучше удалить или поставить AUTO.

Если хотите пробовать имя СОЛНЦАНЕТ для Tele2, можно поставить:

SIGMA_SENDER=СОЛНЦАНЕТ
SIGMA_SENDER_MODE=fallback

Тогда система сначала попробует отправить с СОЛНЦАНЕТ, а при ошибке Sender not found повторит отправку без sender.

Лучший вопрос менеджеру SIGMA:

"При API-отправке без параметра sender будет ли SIGMA автоматически использовать СОЛНЦАНЕТ для Tele2 и ваши общие имена для остальных операторов? Если да, мы будем отправлять без sender. Если нет — дайте точное техническое значение sender/default sender, которое нужно передавать в API для всех операторов."
