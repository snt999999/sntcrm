# SIGMA: ошибка Could not find matching template

Если SIGMA возвращает:

Could not find matching template

значит sender принят, API работает, но текст SMS не совпал с шаблоном, который зарегистрирован в SIGMA.

В этой версии все тексты приводятся к утвержденным шаблонам из Excel:

1. СОЛНЦАНЕТ: запись оформлена на ДД.ММ.ГГГГ ЧЧ:ММ.
2. СОЛНЦАНЕТ: напоминаем о записи ДД.ММ.ГГГГ, ЧЧ:ММ.
3. СОЛНЦАНЕТ: до записи осталось 2 часа.
4. СОЛНЦАНЕТ: запись перенесена на ДД.ММ.ГГГГ ЧЧ:ММ.
5. Спасибо, что выбрали СОЛНЦАНЕТ! Оставьте отзыв: https://clck.su/solncanet

Если ошибка останется, в ответе функции будет поле outgoingText — это точный текст, который ушёл в SIGMA.
Его нужно отправить менеджеру SIGMA и попросить проверить, что в личном кабинете шаблон создан именно с переменными, а не с буквальным текстом {Дата} и {Время}.

Текст для менеджера SIGMA:

У нас sender = Solncanet. API принимает отправителя, но возвращает ошибку Could not find matching template.
Вот точный текст, который уходит в API: <вставить outgoingText>.
Проверьте, пожалуйста, что шаблон в ЛК создан с переменными для даты/времени и соответствует этому тексту.
Нужно, чтобы проходили такие варианты:
- СОЛНЦАНЕТ: запись оформлена на 15.07.2026 10:00.
- СОЛНЦАНЕТ: напоминаем о записи 15.07.2026, 10:00.
- СОЛНЦАНЕТ: до записи осталось 2 часа.
- СОЛНЦАНЕТ: запись перенесена на 15.07.2026 10:00.
- Спасибо, что выбрали СОЛНЦАНЕТ! Оставьте отзыв: https://clck.su/solncanet
